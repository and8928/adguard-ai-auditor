## 🎯 Что это такое

**AdGuard AI Auditor** - это интеллектуальный инструмент на FastAPI, который подключается к вашему серверу AdGuard Home, собирает логи DNS-запросов и проводит их аудит с помощью больших языковых моделей (Google Gemini, Vertex AI и DeepSeek).

![main finish audit.png](photo/main%20finish%20audit.png)

Автоматизирует анализ историю сетевого трафика: находит пропущенные трекеры и рекламу, выявляет ложные срабатывания (когда блокировка ломает легитимные приложения) и предлагает рекомендации по настройке фильтров.

## ⚙️ Раздел настроек

Откройте диалог **⚙️ Настройки** из верхней панели, чтобы настроить приложение на лету — без правки `.env` и перезапуска. Изменения сохраняются в `data/state.env` и сразу применяются к работающему процессу.

![раздел настроек](photo/settings.png)

**Как работает сохранение**

- **Секреты — «только на запись».** Поля пароля и API-ключей показывают заглушку `••• задано` и никогда не возвращаются API — оставьте поле пустым, чтобы сохранить текущее значение, или введите новое, чтобы заменить его.
- **Изменение URL / Порта** пересобирает эндпоинты AdGuard на лету.
- **Изменение URL / Порта / Логина / Пароля** сбрасывает текущую сессию, вынуждая заново войти при следующем запросе.
- **Тест коннект** проверяет, действителен ли текущий `AGH_SESSION`; **Тест логин** выполняет полный повторный вход с сохранёнными учётными данными.

## 🚀 Установка и запуск

Рекомендуемый способ установки и запуска проекта - через **Poetry**.

### 1. Клонирование репозитория
```bash
git clone https://github.com/and8928/adguard-ai-auditor.git
cd adguard-ai-auditor
```

### 2. Установка зависимостей через Poetry
Настройте Poetry, чтобы виртуальное окружение создавалось внутри папки проекта:
```bash
poetry config virtualenvs.in-project true
poetry install
```

### 3. Настройка окружения
Создайте файл `.env` в корне проекта. Шаблон можно скопировать из `.env.example`:
```ini
# Настройки AdGuard
# ADGUARD_BASE_URL по умолчанию http://host.docker.internal (хост Docker).
# При запуске вне Docker укажите адрес вашего AdGuard, например http://192.168.1.1
ADGUARD_BASE_URL="http://192.168.1.1"
ADGUARD_PORT=3333
ADGUARD_USER="your_user"
ADGUARD_PASSWORD="your_password"
ADGUARD_STEP_REQ=100
# Примечание: AGH_SESSION выдаётся автоматически после входа и хранится в data/state.env - вручную задавать не нужно.

# Настройки Google Gemini (опционально)
# Список моделей в виде JSON-строки (парсится Pydantic). Перебираются по порядку.
GEMINI_MODELS_NAME='["gemini-3-pro-preview","gemini-3-flash-preview", "gemini-3.1-pro-preview"]'
GEMINI_API_KEY = "gemini_key"

# Настройки Vertex AI (опционально)
VERTEX_AI_MODELS_NAME='[]'
VERTEX_AI_API_KEY = ""

# Настройки OpenAI (пока не работает)
OPENAI_MODEL_NAME="gpt-5-mini"
OPENAI_API_KEY = "openai_key"

# Настройки deepseek (опционально)
DEEPSEEK_MODELS_NAME='["deepseek-v4-flash","deepseek-v4-pro"]'
DEEPSEEK_API_KEY = "gemini_key"
DEEPSEEK_REASONING_EFFORT = high #low,medium,high
DEEPSEEK_THINKING_ENABLED = True

# Установите True для подробного отладочного логирования
DEBUG_MOD = False
```

> [!NOTE]
> **Настройки в рантайме.** Большинство этих значений (язык, URL/порт AdGuard, логин/пароль,
> шаг запроса, API-ключи LLM и списки моделей по провайдерам) можно менять на лету через панель **⚙️ Настройки** в веб-интерфейсе.
> Изменения сохраняются в `data/state.env` и применяются без перезапуска — файл `.env` задаёт
> лишь начальные значения по умолчанию.

> [!WARNING]
> **Секреты хранятся в открытом виде** в `.env` и `data/state.env` (пароль AdGuard и API-ключи
> LLM). Это осознанное решение для **локально доступного** сервиса: процессу это значение всё
> равно нужно в открытом виде, поэтому шифрование на диске дало бы лишь ложное чувство
> безопасности. Держите развёртывание локальным, убедитесь, что `data/` **не** попадает в git
> (он в `.gitignore`) и **не** зашивается в Docker-образ (он в `.dockerignore`), и ограничьте
> доступ к файловой системе на хосте, если машиной пользуется кто-то ещё.

### 4. Запуск приложения
```bash
poetry run uvicorn src.adguard_auditor.main:app --reload
```

Приложение будет доступно по адресу: `http://localhost:8000`
Документация API (Swagger): `http://localhost:8000/docs`

### 5. Тестирование
В проекте есть набор тестов, покрывающий основную логику, эндпоинты API и интеграции с LLM:
```bash
poetry run pytest tests/ -v
```

> [!TIP]
> Все внешние вызовы (AdGuard Home, API LLM) мокаются, поэтому тесты запускаются без активных инстансов и API-ключей.

## 🐳 Запуск через Docker

В проект включены `Dockerfile` и `docker-compose.yml` для развёртывания в контейнере.

> ### ⚡ Быстрый старт - одна команда, без настройки
> Вставьте это в терминал. Скрипт скачает проект, создаст `.env` по умолчанию, затем соберёт и запустит всё - **не задавая вопросов**:
>
> ```bash
> bash <(curl -fsSL https://raw.githubusercontent.com/and8928/adguard-ai-auditor/main/install.sh)
> ```
>
> После завершения откройте **`http://<ip-сервера>:3334`** и завершите настройку прямо в панели **⚙️ Настройки** (хост/логин AdGuard + ваш AI-ключ). 🎉
> Запускайте ту же команду в любой момент, чтобы пересобрать и перезапустить.

Предпочитаете мастер в терминале? Добавьте `config`, чтобы запустить интерактивную настройку (хост / порт / логин / пароль AdGuard и **какой AI-провайдер вы используете** - ключи сохранятся автоматически). Удобно для headless-серверов или для повторной настройки:
```bash
bash <(curl -fsSL https://raw.githubusercontent.com/and8928/adguard-ai-auditor/main/install.sh) config
```

Уже склонировали репозиторий? Запустите локально:
```bash
chmod +x install.sh
./install.sh          # поднять; настроить в UI
./install.sh config   # или запустить мастер в терминале
```

### 🔄 Обновление

Из склонированного репозитория:
```bash
./install.sh update
```
Скрипт подтягивает свежий код (fast-forward), добавляет в ваш `.env` настройки, появившиеся в новом релизе (существующие значения и ключи не перезаписываются), затем пересобирает и перезапускает контейнер. Ваш `.env` и каталог `data/` сохраняются.

Если в отслеживаемых файлах есть локальные правки, обновление остановится и попросит закоммитить или спрятать их через stash - ничего не будет перезаписано молча.

Хотите настроить всё вручную? Следуйте шагам ниже.

### 1. Подготовка окружения
Создайте файл `.env`, как описано выше. Каталог `data/` (монтируется как volume) хранит рабочее состояние - автоматически выданный `AGH_SESSION` и ваши промпт-правила, поэтому данные сохраняются между перезапусками.

### 2. Сборка и запуск
```bash
docker compose up -d --build
```

Дашборд будет доступен по адресу `http://<ip-сервера>:3334` (проброшен на внутренний порт контейнера `8000`).

> [!NOTE]
> `ADGUARD_BASE_URL` по умолчанию равен `http://host.docker.internal` и указывает на хост-машину Docker - в compose-файле `host.docker.internal` уже сопоставлен с шлюзом хоста. Если AdGuard Home работает на том же хосте, менять ничего не нужно; укажите другой адрес, если AdGuard на другой машине.

## 📡 Эндпоинты API

Основные эндпоинты в пространстве имён `/api/v1` (полный список - в Swagger по адресу `/docs`):

- `GET /` - Отдаёт интерактивный веб-дашборд (корень `/` автоматически перенаправляет сюда).
- `GET /get-raw-request-log?limit=100` - Получить сырые логи запросов с сервера AdGuard Home.
- `GET /get-response-log?limit=100` - Получить очищенные и сгруппированные логи (Allowed / Blocked).
- `POST /ai-analis-data` - Запустить LLM-анализ по переданным данным; модель выбирается параметром `model_services`.
- `POST /auto-analis` - Получить логи из AdGuard, очистить их и прогнать через LLM одним вызовом (не-SSE аналог полного аудита).
- `GET /audit/stream` - SSE-эндпоинт, транслирующий прогресс и результаты аудита в реальном времени. Поддерживает `action=full|fetch|analyze`; также отдаёт оценку токенов на входе (`est_tokens`) и фактический расход модели (`usage`: вход/выход/всего).
- `GET /audit/cache`, `POST /audit/cache/clear` - Просмотр или очистка закэшированных полученных/очищенных данных.
- `POST /to_block`, `POST /to_unblock`, `POST /to_delete` - Применить решения о блокировке/разблокировке или удалить пользовательские правила прямо в фильтрах AdGuard Home.
- `GET /get_actual_filter` - Получить текущие оптимизированные правила пользовательского фильтра из AdGuard Home (питает интерактивный менеджер правил фильтра).
- `GET /prompt-rules`, `POST /prompt-rules`, `GET /prompt-rules/{id}`, `PATCH /prompt-rules/{id}`, `DELETE /prompt-rules/{id}` - CRUD-эндпоинты для управления пользовательскими промпт-правилами и указаниями для ИИ.
- `GET /prompt-rules/{id}/test` - Предпросмотр блока промпта, который правило добавит к системным инструкциям ИИ.
- `GET /settings`, `PUT /settings` - Чтение и обновление настроек в рантайме (URL/порт/логин/пароль AdGuard, шаг запроса, API-ключи LLM и списки моделей по провайдерам). Секреты никогда не возвращаются в `GET` (только флаги `*_set`), а пустой секрет в `PUT` сохраняет текущее значение.
- `POST /settings/test_connection` - Проверить, действителен ли текущий `AGH_SESSION`.
- `POST /settings/test_login` - Проверка входа в AdGuard Home с текущими сохранёнными учётными данными.
